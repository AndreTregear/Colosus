## 2. New Research Summary

### 2.1 Payment Collection Infrastructure (technology/15)

**The core problem:** 75-90% of Yaya's target users don't have credit cards. Traditional SaaS billing is DOA.

**Solution landscape:**

| Method | Adoption | Recurring? | Best Phase |
|---|---|---|---|
| Yape (via gateway) | 75-85% of target | Yes (new 2025) | Phase 2+ |
| WhatsApp QR + manual | 95% | No (manual) | MVP/Phase 1 |
| Agente BCP (cash) | ~95% | No | Fallback |
| Contador wholesale | N/A (B2B) | Yes (invoice) | Phase 3+ |
| Debit card | 35-45% | Yes | Phase 4 |

**Breakthrough finding:** Yape now supports recurring on-file payments (EBANX partnership, July 2025). 93% approval rate. This is Yaya's primary billing channel.

**Killer insight:** The contador channel solves BOTH distribution AND billing. Instead of collecting S/29/month from 300 individual salon owners, collect S/660/month from 15 contadores (20 users each × S/22 wholesale). One invoice, one payment, zero individual collection headaches.

**Recommended gateway:** Monnet Payments (Pay-me) — Peru-native, has Yape subscription API with create/cancel/webhook.

### 2.2 Voice AI Fine-Tuning Strategy (technology/16)

**The accuracy challenge:** Whisper has 11,100 hours of Spanish training data, but Peruvian Spanish (especially informal working-class Lima) is underrepresented. Financial numbers are the critical failure mode — "ochenta" (80) vs "sesenta" (60) is a single phoneme difference.

**Fine-tuning plan:**
- Base model: whisper-large-v3 (1.55B params)
- Fine-tuning method: LoRA/PEFT (trainable on single A5000)
- Training data: 15-30 hours (synthetic + public + real recordings)
- Framework: faster-whisper for inference (4x speed, int8)
- Training time: ~10-20 hours on c.yaya.sh

**Target accuracy thresholds:**
- Overall WER: <10%
- Financial WER: <5%
- Number Exact Match: >95%
- Critical Error Rate: <2%

**Two-stage accuracy pipeline:** Whisper transcription → LLM structured extraction (Qwen3.5-27B). Redundant verification on financial data.

**Cost advantage quantified:** Self-hosted Whisper = ~$50-100/month vs. cloud ASR at $900-7,200/month at 1,000 users.

### 2.3 Post-Launch KPI Dashboard (strategy/20)

**North Star metric:** Voice notes per active user per day (VNPUAD). Target: 3-7/day.

**Core metrics framework:**
- Tier 1 (Daily): VNPUAD, WAU, Transcription Accuracy Rate
- Tier 2 (Weekly): Revenue, Time to First Voice Note, Day 7/30 Retention, Payment Success Rate
- Tier 3 (Monthly): Contador Acquisition Rate, Organic Referrals, Feature Adoption, NPS

**PMF signals (behavioral, no survey needed):**
- Day 30 retention >50% ✅
- Voice notes/day >5 for power users ✅
- Organic referrals >20% of new users ✅
- Users screenshot and share daily summaries ✅
- Users resist cancellation when payment fails ✅

**Key innovation:** Daily dashboard delivered via WhatsApp to Andre at 8:00 AM. No web admin panel needed for first 100 users.

**Anti-metrics:** CAC, LTV, MRR growth rate, traffic, app downloads — all meaningless with <50 users in the first 90 days.

## 3. Cross-Document Insights (Cycle #16)

### 3.1 The Payment-Accuracy-Trust Triangle

Connecting all three new documents reveals a critical interdependency:

```
Payment Collection ←→ Transcription Accuracy ←→ User Trust
         ↑                                            ↓
         └──────────── Retention/Revenue ──────────────┘
```

- If transcription accuracy is low → users don't trust the data → they don't see value → they don't pay
- If payment collection fails → users churn involuntarily → KPIs show false churn signal
- If KPIs aren't tracked → Andre can't distinguish accuracy problems from payment problems from value problems

**All three must work simultaneously for Yaya to function.** This is why all three needed to be researched before coding begins.

### 3.2 The Contador as Universal Solution

Across all 86 documents, the contador (accountant) channel emerges as the single most powerful strategic lever:

| Problem | How Contador Solves It |
|---|---|
| Distribution | Each contador brings 15-30 pre-qualified users |
| Payment collection | Single monthly invoice replaces 30 individual payments |
| Support | Contador provides first-line support for their clients |
| Trust | Salon owner trusts their contador's recommendation |
| Churn | Contador has incentive to keep users active (margin) |
| Compliance | Contador ensures tax data is correct |
| Scaling through "valley of death" | Removes Andre's per-user support burden |

**The strategic implication:** Yaya should be designed as much for the contador as for the salon owner. The contador dashboard/view may be MORE important than the individual user experience for scaling.

### 3.3 Voice Notes Per Day as the Universal Predictor

Strategy/20 identifies VNPUAD as the north star metric. This connects to:
- technology/16: Higher accuracy → more voice notes (trust)
- strategy/19: More voice notes → more data → better reports → higher retention → revenue
- strategy/16: More data → stronger data moat → competitive defensibility
- market/15: December voice note surge → seasonal revenue validation

**Every strategic objective in Yaya's business plan ultimately depends on whether salon owners send voice notes.** This simplifies Andre's focus enormously: build the thing that makes people want to send voice notes, and make sure every voice note is transcribed accurately.

## 7. Top Insights for Andre (Cycle #16 — The Final Three)

### 1. Start Billing with WhatsApp QR + Manual Yape (Cost: $0)
For Customer Zero and the first 10 users, don't integrate a payment gateway. Send a Yape QR code via WhatsApp monthly. Manually check for incoming payments. This works perfectly at small scale and costs nothing. Integrate Pay-me's Yape subscription API only when approaching 50 users.

### 2. Build the Whisper Fine-Tuning Pipeline in Week 1-2
Before any user sends a voice note, Yaya needs >92% transcription accuracy on Peruvian Spanish financial terms. This means:
- Week 1: Generate 200 synthetic financial utterances, create fine-tuning dataset
- Week 2: Fine-tune whisper-large-v3 with LoRA on c.yaya.sh (~10-20 hours training)
- Test: Verify Number Exact Match >95% before Customer Zero onboarding
- Ongoing: Every corrected transcription becomes training data

### 3. Your Daily Dashboard Is a WhatsApp Message, Not a Web App
At 8:00 AM every day, Yaya should WhatsApp you a 10-line summary: active users, voice notes yesterday, accuracy rate, revenue recorded, who's inactive, what needs attention. Don't build an admin panel. Don't set up Metabase. The dashboard IS the WhatsApp message.

### Bonus: The One Metric That Matters
If salon owners send 3-7 voice notes per day without prompting by Day 7, Yaya has product-market fit. Everything else — revenue, retention, NPS, contadores — will follow from that single behavior.
